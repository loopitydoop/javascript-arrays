About Javascript Arrays

Arrays have the "object" data type. But they are a special kind of object that operates very differently from regular objects, so it's better that you think of them as their own thing

Key Features of Arrays

Arrays store multiple values

You can think of Arrays as lists of values (Like, I think they're really similar to Python Lists). Arrays can contain multiple values. Consider the following array, pets

pets = ["dogs", "cats", "hedgehogs"];

pets[0] would be "dogs"
pets[1] would be "cats"
pets[2] would be "hedgehogs"

Array elements are numbered and ordered

Each element in an array has an index number, starting from 0, and ascending by 1 for each new element. In that sense they're basically arranged in numerical order.

And I guess as a consequence of having a sequence, you can loop through arrays using for loops.

How to Make an Array

Using an Array Literal

Basically if you surround a comma-separated list with square brackets, that's an "Array Literal"

testArray = ["pork", "chicken", "beef"];

You Don't Have to Provide Any Elements First

you can just declare an empty array first:

const meats = [];

then afterwards you can add values to the array:

meats[0] = "chicken"

meats[1] = "beef"

Using the Javascript Keyword “new” (seems kinda jank and not useful)

const clothes = new Array("hats", "shirts", "pants", "shoes");

w3schools says that “for simplicity, readability and execution speed, use the array literal method.” instead of the Javascript keyword “new”, and that they both do exactly the same thing in practice. So it sounds to me like this "new" keyword isn't really useful for making arrays.
